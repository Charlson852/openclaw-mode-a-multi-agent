# 默认分发规则与当前落地状态

## 一、默认分发规则

在老板未明确指定执行者时，`main` 默认按以下规则调度：

- **analyst**：A股复盘、选股、盘前计划、题材/龙头/个股研究、量化与结构化判断
- **publisher**：博客、摘要、对外表达、发布整理、文案交付
- **ops**：服务器、网关、cron、Docker、宝塔、反代、配置、系统排障
- **reviewer**：复核、审阅、查遗漏、关键结果验收
- **monitor**：巡检、告警、结果落地检查、异常发现

`main` 默认职责：
- 接收需求
- 拆解任务
- 路由到对应 agent
- 汇总结果
- 保留短链路/应急任务的直接执行权

## 二、当前已经落地的链路

### 1. 分析链（已开始接管）
- A股中台日报 cron 已明确按 analyst 口径执行
- 主链要求：真实跑脚本、真实发送 txt/xlsx、写 sent 标记

### 2. 发布链（已开始接管）
- 博客发布 cron 已明确按 publisher 口径执行
- 主链要求：真实写文件、真实 git push、成功后再通知用户

### 3. ops 线（已开始接管）
- 仓库同步 cron 已明确按 ops 口径执行
- 模型同步清理 cron 已明确按 ops 口径执行
- 宝塔 / 反代 / 服务 / 配置类问题已纳入 ops 默认职责

### 4. 复核链（已开始接管）
- A股中台复核任务已明确仅审 A股中台日报链，不再误审博客链
- 运维修复复检任务已明确命名为 `ops recovery reviewer-check`

### 5. 巡检链（已开始接管）
- monitor 巡检任务已明确命名为 `ops monitor patrol`
- monitor 脚本已收敛到真正仍在使用的链路：
  - 中台发送标记
  - 当天博客文件
  - gateway 状态
  - 高优先级日志错误
- 已移除对已下线 7891 看板的误报检查
